• CGI is specified to

    • start an executable, passing parameters
    • wait for the executable to terminate
    • send output back to HTTP client
  • Problems with this for Smalltalk

    • Smalltalk executables (images) usually large - anywhere from 1/2 Meg to 15 Meg.

    • Multi-second initialization time.

    • Unable to share image resources across programs running in parallel (for some implementations)
